# 《再睡五分钟》H5 小游戏设计规格

## 1. 产品概述

《再睡五分钟》是一款面向手机竖屏的 3–5 分钟 H5 互动小游戏。玩家被告知自己是一名“封印师”，必须阻止一只沉睡巨兽苏醒。玩家通过点击、拖动和画线，消除卧室中的声音、光线和震动。

随着游戏推进，任务提示逐渐暴露出不合常理的信息。结尾镜头拉远，揭晓所谓“封印系统”其实是手机闹钟界面，玩家的真实身份只是“稍后提醒”按钮。系统一本正经地宣布世界获救，而赖床的上班族因迟到失去全勤奖。

核心体验由三部分组成：即时操作爽感、逐步升级的规则异常，以及带有职场讽刺意味的荒诞身份反转。

## 2. 设计目标

- 玩家无需阅读长说明，首次接触后 10 秒内理解基本操作。
- 单局时长稳定在 3–5 分钟，完整体验教学、升级、高潮和反转。
- 点击、拖动、画线三种操作均自然服务于场景，不像孤立小游戏合集。
- 失误不会阻止玩家看到结局，只影响评分、评语和扣款结果。
- 美术、动画和文字共同营造夸张漫画式爽感与荒诞喜剧感。
- 首版保持单局体验，不加入金币、体力、道具、广告、排行榜或养成系统。

## 3. 目标平台与显示方式

- 运行环境：手机浏览器和微信内置浏览器。
- 屏幕方向：竖屏；横屏时暂停游戏并显示旋转手机提示。
- 设计基准：375 × 667 CSS 像素，使用等比缩放和安全区适配覆盖常见竖屏尺寸。
- 输入方式：单指触控。首版不要求多指手势，也不使用重力感应。

## 4. 核心体验结构

游戏始终发生在同一间卧室。镜头以快速推近、横移和拉远的方式聚焦不同物品，使各阶段保持空间连续性。

体验节奏为：

1. 前段通过单一点击操作建立理解与爽感。
2. 中段依次引入拖动和画线，增加操作变化。
3. 后段交替组合三种操作，提高密度并让提示语露出破绽。
4. 高潮结束后先进行史诗式庆祝，再通过镜头拉远完成身份反转。
5. 结算页展示评级、迟到结果和重新挑战入口。

## 5. 关卡流程

### 5.1 闹钟入侵（0:00–0:40）

神秘旁白提示：“沉睡巨兽即将苏醒，立刻摧毁声波装置。”镜头聚焦床边闹钟。

- 玩家连续点击闹钟将其拍灭。
- 越接近响铃，闹钟的抖动、红光和铃声频率越强。
- 命中后闹钟夸张压扁，屏幕轻微震动，并弹出“啪！”或“闭嘴！”等拟声词。
- 未及时处理时，床上角色翻身，记为一次失误，但任务继续。
- 本阶段承担点击教学，不同时引入其他手势。

### 5.2 封锁晨光（0:40–1:25）

提示：“封锁天穹裂隙，禁止晨光接触巨兽。”镜头推向窗户。

- 玩家分别拖动左右窗帘，直到两侧在中央闭合。
- 窗帘使用明显的弹性拉伸和松手回弹，强化拖动手感。
- 飞鸟、车辆反光等干扰会短暂扯开窗帘，玩家需要再次拉合。
- 阳光持续照到床铺超过宽限时间时记为一次失误，随后自动进入下一次机会。
- 本阶段承担拖动教学。

### 5.3 静音结界（1:25–2:20）

提示：“绘制静音结界，隔绝异界通讯。”镜头在手机、门铃和扫地机器人之间移动。

- 玩家围绕发声物体画出一条闭合轨迹。
- 轨迹首尾进入容差范围即视为闭合，不要求形状准确。
- 成功后轨迹膨胀为漫画气泡，声音拟声词被吸入并消失。
- 轨迹未闭合或超过处理时间记为一次失误，系统清除轨迹并继续。
- 本阶段承担画线教学。

### 5.4 全面入侵（2:20–3:30）

提示与场景同时加速，点击、拖动和画线任务交替出现。

- 同一时刻最多出现两个需要处理的目标，避免触屏区域重叠。
- 操作类型以短序列轮换，保证玩家能判断目标所需手势。
- 提示逐渐出现异常内容，例如：“再坚持 60 秒，他的全勤奖马上消失！”
- 危机值随失误增长，床上角色会翻身、皱眉或短暂坐起，但不会触发提前结束。
- 最后 20 秒达到最高操作密度，随后所有声音突然停止。

### 5.5 身份揭晓（3:30–4:20）

- 画面播放夸张的光芒、礼花和“世界已获救”字幕。
- 镜头逐层拉远，法阵边框变成手机闹钟界面。
- “封印完成”按钮变为“稍后提醒”，显示时间已向后推迟五分钟。
- 上班族惊醒，发现迟到；通知栏显示全勤奖扣款信息。
- 游戏根据累计失误展示 S、A 或 B 评级。

阶段时间是目标值而非逐帧硬限制。具体实现可在内部调节生成间隔，使正常玩家总时长保持在 3 分 30 秒至 4 分 40 秒，并确保极端表现下仍不超过 5 分钟。

## 6. 失败与评分规则

游戏使用软失败：玩家不会因失误重开或错过反转。

- 失误定义：未在规定时间内处理目标，或画线未闭合。
- 每次目标只计一次失误，避免同一目标持续超时造成重复扣分。
- 失误只影响危机演出和结算内容，不改变主流程长度。

评分区间：

- S：0–2 次失误。评语：“完美封印，全勤奖彻底消失。”
- A：3–6 次失误。评语：“世界获救，但他差点醒了。”
- B：7 次及以上失误。评语：“勉强多睡五分钟，扣款照常发生。”

扣款金额采用预设的娱乐化数值，与评分等级对应，不模拟真实工资计算。结算页提供“再睡一次”按钮，点击后直接重置本局状态并重新开始。

## 7. 界面布局

### 游戏中

- 顶部：单行任务提示；只呈现当前目标，避免长文字遮挡场景。
- 中部：卧室与互动对象，占据主要可视区域。
- 侧边：竖向“苏醒危机值”，失误时上升并抖动。
- 底部：仅在新手势首次出现时显示短操作提示，成功一次后隐藏。
- 系统区域：避开刘海、圆角和底部手势条的安全区。

### 结算页

- 评级字母是第一视觉焦点。
- 展示荒诞评语、迟到分钟数和全勤奖扣款金额。
- 主按钮为“再睡一次”。
- 首版不提供登录、排行或强制分享入口。

## 8. 视觉与动画

视觉方向采用夸张漫画风：

- 使用粗描边、鲜艳高对比色和简化几何造型。
- 卧室物品允许明显拉伸、压扁和回弹，不追求写实物理。
- 成功反馈使用短促拟声词、速度线和局部震屏。
- 失误反馈通过角色翻身、皱眉和危机条抖动表达，不使用惩罚性红屏。
- 反转前使用史诗法阵和庆祝特效；拉远后迅速回到普通手机界面，形成视觉落差。
- 低性能模式减少粒子数量、残影和震屏幅度，但保留关键形变与拟声词。

## 9. 音频与触觉反馈

- 主要音频包括闹钟、呼噜声、窗帘摩擦、手机震动、门铃和漫画打击音。
- 音频必须在玩家首次触摸后统一解锁，以兼容微信和移动浏览器限制。
- 无声或音频加载失败时，所有关键信息仍通过动画、图标和文字表达。
- 支持设备时，在成功点击和高潮阶段使用轻量振动；不支持振动时静默降级。
- 页面切到后台时暂停计时和音频，返回后从暂停点继续。

## 10. 技术设计

首版使用 TypeScript 和 Canvas 2D。DOM 负责页面容器、方向提示和必要的可访问文本；游戏场景、动画和触控命中主要由 Canvas 渲染。

核心模块：

1. **场景控制器**：管理五个阶段的进入、更新、退出和镜头切换。
2. **输入系统**：把浏览器触摸事件统一识别为点击、拖动和画线轨迹。
3. **任务系统**：生成目标、管理处理时限并判断成功或失误。
4. **反馈系统**：播放形变、拟声词、震屏、音效和振动。
5. **评分系统**：累计失误并生成评级、评语和扣款结果。
6. **资源加载器**：预加载关键图片与音频，提供失败降级和加载进度。
7. **生命周期管理器**：响应前后台切换、屏幕方向和页面尺寸变化。

主要数据流：

`玩家触摸 → 输入识别 → 当前任务判定 → 动画/音效反馈 → 更新失误与阶段进度 → 场景控制器切换阶段`

模块边界要求：场景只声明目标与节奏，不直接监听原始浏览器事件；输入系统不理解关卡业务；评分系统不负责播放动画。这样可以独立测试手势识别、任务判定和评分边界。

## 11. 资源加载与异常处理

- 首屏仅加载启动所需的字体、卧室基础图和第一阶段音效，其余资源在教学阶段后台加载。
- 关键资源失败时使用纯色几何占位和系统字体，游戏仍可完成。
- 次要音效失败时跳过音效，不阻塞阶段切换。
- 首屏加载超时后显示重试按钮；已经进入游戏后不因单个资源失败强制重载。
- 触控取消、浏览器失焦或方向变化时，结束当前手势并恢复到可重新操作状态。
- 页面滚动、双击缩放和游戏区域内的浏览器默认触摸行为应被阻止。

## 12. 测试范围

### 单元测试

- 点击、拖动与闭合画线的识别阈值和边界情况。
- 单个目标超时只记录一次失误。
- S/A/B 评分在 2、3、6、7 次失误处的边界。
- 暂停与恢复时阶段计时不跳变。
- Canvas 坐标与不同屏幕尺寸之间的映射。

### 集成测试

- 五个阶段按顺序进入和退出。
- 资源加载失败时可以使用降级资源完成整局。
- 前后台切换、横竖屏切换后可以继续游戏。
- 静音模式下关键信息仍完整可理解。
- 结算后“再睡一次”彻底重置失误、计时和场景状态。

### 设备验证

- 微信内置浏览器中的主流 iOS 和 Android 设备。
- 较小竖屏、刘海屏和高纵横比设备。
- 中低性能设备在高潮阶段的帧率与触控响应。

## 13. 首版验收标准

- 玩家无需安装或登录即可开始游戏。
- 正常网络环境下，进入页面后数秒内出现可操作内容；完整资源可继续后台加载。
- 三种操作在常见触屏设备上能够稳定识别，并有明确成功与失误反馈。
- 一局总时长为 3–5 分钟。
- 所有玩家都能看到完整身份反转和结算页。
- 音频关闭、单个非关键资源失败、前后台切换或横屏误入不会卡死流程。
- S/A/B 评分严格按累计失误次数生成。
- 结算后可以立即重新挑战，且上一局数据不会泄漏到新一局。

## 14. 首版明确不包含

- 多结局和分支剧情。
- 隐藏成就、收藏品或每日任务。
- 登录、云存档、好友排行和服务端接口。
- 金币、道具、体力、抽奖和付费内容。
- 强制分享、广告接入和数据运营后台。
- 横屏专用布局与桌面端专用交互。

这些能力可在核心单局体验经过试玩验证后另行设计，不应进入首版实现计划。
